# チケットステータス管理

## ステータス（ラベル）の定義

| ラベル | 意味 | トリガー |
|--------|------|----------|
| 検討中 | 作った直後。内容を検討中 | チケット作成時 |
| 実装待ち | 承認済み。ユーザーが「実装して」と指示するのを待っている | ユーザーが承認したが、まだ実装指示を出していない |
| レビュー待ち | ステージングにデプロイ済み。確認待ち | 実装完了・staging プッシュ後 |
| 完了 | メインにデプロイ済み | main マージ・本番デプロイ後 |

※「完了」は Issue を **クローズ** して表現する（ラベルではなく）

## フロー

```
検討中 → 実装待ち → [実装] → レビュー待ち → 完了(close)
   ↑         ↑                    ↑            ↑
 作成時   承認時               staging デプロイ  main デプロイ
```

## GitHub デフォルトでできること

- **ラベル**: 自由に作成・付与可能。上記 4 段階をラベルで表現できる
- **Open/Closed**: 完了 = Close で表現

## ギャップ（デフォルトの制約）

1. **複数ラベル**: 1 Issue に複数ラベルを付けられる。ステータスは「1つだけ」にしたいので、**付け替え運用**が必要（実装待ちにしたら検討中を外す等）

2. **自動化**: 以下は手動 or GitHub Actions で実装が必要
   - ステージングデプロイ成功時 → レビュー待ち に付け替え
   - main マージ時 → クローズ

3. **相互排他**: ラベル同士の排他は GitHub にはない。運用で「今のステータス」を1つに保つ

## 推奨ラベル作成

```bash
gh label create "検討中" --color "FBCA04" --description "作った直後・検討中"
gh label create "実装待ち" --color "0E8A16" --description "承認済み・実装指示待ち"
gh label create "レビュー待ち" --color "1D76DB" --description "ステージングデプロイ済み・確認待ち"
# 完了 = close のためラベル不要。既存の done は廃止 or クローズ時のみ付与して即 close も可
```

## 既存ラベルとの対応

| 旧 | 新 |
|----|-----|
| pending-review | 検討中 または レビュー待ち（文脈による） |
| approved | 実装待ち |
| in-progress | （実装中はラベルなし or 実装待ちのまま） |
| done | 完了 = Close（ラベルは不要） |
