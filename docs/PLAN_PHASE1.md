# 開発ステップ概要（Phase 1〜）

## ゴールイメージ

- 「エンデュランスレース一覧」画面が、Supabase 上のデータを元に動く
- 最低限の検索・絞り込み（種別 / 日付）
- データは収集プログラム（の最初のスケルトン）から投入される

---

## Phase 1: 一覧の中身を Supabase 化する

### 1-1. Supabase クライアントと型の整備

- `src/lib/supabaseClient.ts` を作成
- `events` テーブル用の型（TypeScript）を定義
- 将来 categories 等を足せるように最低限の構成にしておく

### 1-2. フロントエンドから Supabase の `events` を読む

- `src/App.tsx` のモックデータを、Supabase からのフェッチに置き換え
- ローディング / エラー表示を追加
- 既存のフィルタ（種別 / 開催月）はそのまま活かす

### 1-3. 初期データ投入用の「収集スクリプト（スケルトン）」

- まずはローカルの JSON など **固定ソース** から Supabase `events` へ INSERT するスクリプトを用意
- 将来のスクレイピング / API 呼び出しの置き換えを前提に、できるだけ「入口だけ差し替えればよい」構成にする

---

## Phase 2: 絞り込み・UX の強化

- レース種別 / 日付以外のフィルタ項目を増やす（例: 県、獲得標高レンジ 等）
- ソート（開催日順、距離順など）
- 一覧から詳細画面への遷移（最低限の詳細ビュー）

---

## Phase 3: 収集プログラムの本格化

- ランネット / 各種公式サイトを対象にした収集処理の設計・実装
- Google / Maps / Directions 等を使ったアクセス情報の取得
- 定期実行（スケジューリング）の設計（Supabase Functions / 外部ジョブ等）

---

## チケット運用について

- すべての実装は **「まずチケットを作成 → 実装 → staging へデプロイ → ユーザー確認」** の流れで進める
- フィードバック（見た目・仕様変更など）も、必ず GitHub Issue を切ってから対応する
- 今後の Phase は、必要に応じて細かくチケットに分割していく
