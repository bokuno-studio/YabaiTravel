# 過去開催（去年のレース）紐付け仕様

## 背景

- 今年のコースマップはレース終了後に公式から消える
- 来年同じ Mt. Fuji のページを作っても、去年のコースマップは公式から探せない
- 内部で「去年のレース」と紐づけ、過去のコースマップ等を参照したい

## 制約

- **バックエンドはクローンで動く**。登録プロセスに人が入るのはありえない
- **9割方正しく自動で紐づく**状態を目指す
- たまにエラーがあっても、**ユーザーが修正提案して承認されたら反映**する方針でカバー（紐づけに限らず全般）

---

## Phase 1: 「去年のレースはこちら」リンク（一旦）

### データ設計

| フィールド | 型 | 説明 |
|------------|-----|------|
| `events.previous_edition_url` | text | 去年のレースへのURL。自サイトでも外部でも可 |

### 表示

- 「去年のレースはこちら」リンクのみ表示
- 修正も簡単（変更リクエストで `previous_edition_url` を提案）

### 自動紐付け（クローン）

新規 event 登録時に、既存 events から「同じレースの去年版」を推定し、`previous_edition_url` を設定する。

**推定ロジック（案）**

1. **大会名の正規化**: 空白・記号除去、年表記除去（"Mt. Fuji 100 2025" → "mtfuji100"）
2. **場所**: location が一致または包含関係
3. **開催時期**: event_date が1年以内の差（例: 今年7月 → 去年6〜8月の event を候補に）
4. **スコアリング**: 上記を組み合わせて類似度スコアを算出、閾値以上なら紐付け

**設定する URL**

- 自サイトの event 詳細: `https://yabai.travel/events/{event_id}` または環境に応じたベースURL
- 外部の場合はそのまま（公式アーカイブ等があれば）

## Phase 2: リンク先からデータ取得してマージ表示（理想）

### 方針

「リンク先から自動でデータ引っ張る」ページにする。**リンク変更になったら取得元が変わるだけ**で済む。

### 自サイトの URL の場合

`previous_edition_url` が自サイトの event 詳細（例: `https://yabai.travel/events/{event_id}`）の場合:

1. URL から `event_id` をパース
2. API（Supabase または内部 API）でその event のデータを取得
3. 取得したデータの一部（コースマップ、申込期間、料金等）をマージして表示

**マージして表示する項目（例）**

- コースマップ（course_map_files）
- 例年の申込期間（entry_start_typical, entry_end_typical）
- カテゴリ別申込費（同一カテゴリ名でマッチ）

### 外部 URL の場合

- リンク表示のみ（Phase 1 と同様）

### 変更リクエストとの連携

- `previous_edition_url` が間違っている場合、ユーザーが正しい URL を提案
- 承認後、次回表示から新しい URL が取得元になる

---

## event_series との関係

- **event_series**: 同一レースの複数年版を「グループ」として扱う概念。既存。
- **previous_edition_url**: 「去年の1件」への直接リンク。シンプルで修正しやすい。

**選択肢**

| 方式 | メリット | デメリット |
|------|----------|------------|
| previous_edition_url のみ | シンプル、外部URLも可、変更リクエストで修正容易 | 2年前以前は辿れない |
| event_series を維持 | 複数年版を一括管理、過去一覧が作りやすい | 紐付けロジックが複雑 |
| 併用 | previous_edition_url で「去年」を直接指定しつつ、event_series で過去一覧も | 二重管理 |

**推奨**: 一旦は `previous_edition_url` のみで進め、必要になったら event_series を活用する。

## 実装タスク（Phase 1）

1. マイグレーション: `events.previous_edition_url` 追加
2. 型定義: Event に `previous_edition_url` 追加
3. CategoryDetail: 「去年のレースはこちら」リンク表示
4. クローン収集ロジック: 新規 event 登録時の自動紐付け（推定ロジック実装）
5. change_requests: `previous_edition_url` を対象フィールドに追加（既存の target_field で対応可）
