## 概要

YabaiTravel は以下のカテゴリで外部サービスと連携しています。

## 7. キー管理・セキュリティガイドライン

### 7.1 キーの取り扱い

| 点 | ルール |
|-----|--------|
| **秘密キー** | `.env.local` のみ。Git に commit しない。`.gitignore` で除外済み |
| **フロント用キー** | `VITE_` プレフィックス。機密性低い（公開文字列と同等） |
| **サーバー用キー** | `SUPABASE_SERVICE_ROLE_KEY` 等。サーバーサイドのみで使用。フロントに露出禁止 |
| **共有方法** | 1Password 等パスワードマネージャー経由。Slack・メール厳禁 |

### 7.2 キー更新時の手順

1. **キー再生成**: 外部サービスの管理画面で新キーを生成
2. **テスト環境で検証**: ローカル `.env.local` で テスト
3. **本番更新**: CI/CD 秘密変数を更新（GitHub Secrets）
4. **旧キー削除**: 外部サービス管理画面で旧キーを削除
5. **ドキュメント更新**: このファイルに状態を記載

### 7.3 キー監視・監査

- **月1回**: Anthropic Usage API で token 残量確認（Dispatcher 実行）
- **随時**: Google Cloud Budget アラート（超過時は自動メール）
- **随時**: Tavily credit 監視（月初リセット）
- **四半期1回**: キー一覧の整理・不要キーの削除

## 8. よくある質問（FAQ）

### Q: 新しいサービスを追加したい場合は？
**A**: 以下の手順で：
1. 責任者会議で用途・コスト・リスクを判断
2. 外部サービスでキー生成
3. `.env.example` に追加（コメント付き）
4. `.env.local` にキー値を記載
5. このファイルを更新
6. Git commit・push

### Q: キーが漏洩した場合は？
**A**: 直ちに：
1. 外部サービスで該当キーを削除・再生成
2. GitHub Secrets を更新
3. CI/CD 環境変数を更新
4. `.env.local` を更新
5. Dispatcher へ報告

### Q: キーの有効期限は？
**A**: 大部分が無期限。ただし：
- **Supabase Access Token**: 有効期限あり（確認必須）
- **Square Access Token**: 定期更新推奨（セキュリティベストプラクティス）
- **Anthropic API Key**: 無期限だが、Token 管理あり

### Q: 複数環境（ステージング・本番）でキーを分ける必要は？
**A**: 現在は本番環境のみ。ステージング は `.env.local` の `VITE_SUPABASE_SCHEMA=yabai_travel_staging` で対応可能（将来検討）。
