# デプロイ・環境運用

## 前提

- Supabase / Vercel の**初期設定はユーザーが実施**
- スキーマ定義はユーザーは関与せず任せる。作りながら変更してよい
- 設定後の**運用・コントロールは任せる**（スキーマ、環境、デプロイフロー等）

---

## 環境構成

| 環境 | 用途 | ブランチ |
|------|------|----------|
| 検証環境（staging） | ユーザー確認用 | `staging` |
| 本番環境（production） | 本番 | `main` |

---

## デプロイフロー

1. **検証へデプロイ** … `staging` ブランチにマージ or プッシュで検証環境へ反映
2. **ユーザー確認** … 検証環境で動作・表示を確認
3. **本番へデプロイ** … ユーザーが「いいよ」と言ったら `main` にマージし、本番へ反映

---

## スキーマ・DB

- **スキーマ定義の自動反映** … マイグレーション等で Supabase に反映
- **他アプリとの共存** … Organization / Project は共通利用。**スキーマで分離**して各アプリを管理する（例: `yabai_travel` スキーマ）
- **作りながら変更** … スキーマは開発中にどんどん変えてよい。ユーザーは関与しない前提で任せる

---

## 運用ルール

- 検証で確認 → OK で本番、という流れを守る
- スキーマ変更時も同様に検証で確認してから本番へ
