# 生成AI利用方針（試行錯誤中）

データ収集等で生成AIを使う想定。テストスクリプトでトライ＆エラーしながら、最も良い方法を採用する。

## 背景

- Gemini API + Google Grounding は便利だが課金額が大きくなった経験あり
- コストを抑えつつ、適切な方法を探りたい

## 検討中のアプローチ

| アプローチ | 概要 |
|------------|------|
| 構造化抽出 | スクレイピングで取得したHTMLを、軽量モデルでJSON等に変換 |
| LLMは補助のみ | 正規表現で取れる部分は取る。曖昧な部分だけLLMに投げる |
| ローカル/軽量モデル | Ollama + Llama 等 |
| 安価なAPI | Claude Haiku、Gemini Flash 等 |
| キャッシュ | 同じページは再解析しない |

## テスト方針

- テストスクリプトを作り、モデル・プロンプト・入力範囲を変えて比較
- コスト・精度・速度のバランスで採用を決める
